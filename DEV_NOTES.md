# Developer Notes — Tifinagh Dictionary API

## Architecture

```
┌─────────────┐   HTTP    ┌──────────────────────────┐
│  Client(s)  │ ────────→ │  Hono App (index.ts)     │
│  (Discord   │ ←──────── │                          │
│   bot, curl,│           │  /help  → JSON listing   │
│   web app)  │           │  /retype → transliterate │
└─────────────┘           │  /translate → lookup     │
                          │                          │
                          │  retype.ts               │
                          │  dictionary.ts (in mem)  │
                          └──────────────────────────┘
```

## File-by-File Breakdown

### `src/index.ts` — Entry Point

Creates a Hono app with CORS enabled, registers three routes, loads the dictionary on startup, and starts `Bun.serve()`.

**Server config**: port defaults to `3001`, overridable via `$PORT`.

**Dictionary loading**: calls `loadSync()` at import time. If no dataset file exists, the server still starts but `/translate` returns 503.

### `src/retype.ts` — Transliteration Engine

Wraps two npm packages:

| Package | Pairs | Direction |
|---------|-------|-----------|
| `abjad-convert` | Arabic ↔ Tifinagh | Direct phonetic mapping |
| `tamazight` | Latin ↔ Tifinagh | IRCAM standard mappings |

For Arabic ↔ Latin (no direct converter available), the engine chains through Tifinagh:
- Arabic → Tifinagh (abjad-convert) → Latin (tamazight)
- Latin → Tifinagh (tamazight) → Arabic (abjad-convert)

**Script detection**: Uses Unicode range checks for Tifinagh (`\u2D30-\u2D7F`) and Arabic (`\u0600-\u06FF`). Falls back to Latin for everything else.

### `src/dictionary.ts` — Dataset Loader + Search

**Loading**: reads `data/dictionary.json` (generated by the download script) and builds three in-memory search indexes:
- `byAmazigh` — Tifinagh word → entries
- `byArabic` — Arabic translation → entries
- `byEnglish` — English translation → entries

**Search strategy** (`fuzzyMatch`):
1. Exact match (case-insensitive)
2. Starts-with
3. Contains

Results are ordered by match quality (exact first, then prefix, then contains). Maximum 10 results returned.

**Data format**:

```typescript
interface DictEntry {
  word: string;          // Tifinagh
  pronunciation: string; // Latin (IPA or phonetic)
  type: string;          // Part of speech (from dglai)
  arabic: string;        // Arabic translation(s)
  english: string;       // English translation(s)
}
```

### `scripts/download-dataset.ts` — Dataset Downloader

Fetches data from Hugging Face's Datasets Server API (`datasets-server.huggingface.co/rows`).

**Parallelism**: All pages for each dataset are fetched concurrently via `Promise.all`.

**Exclusion**: Only entries with Arabic or English are kept. Entries that only have French are dropped entirely.

**Output**: Writes to `data/dictionary.json` as a JSON object with two top-level keys (`amawal` and `dglai`), making the source of each entry traceable.

## Dataset Licensing

- **amawal-dataset**: CC-BY-4.0 — free to use, share, adapt, even commercially
- **ircam-dglai-dataset**: Based on IRCAM's DGLAi. May require permission from IRCAM for commercial use. The API code is Unlicense, but users are responsible for complying with dataset licenses

The `data/` directory is gitignored; datasets must be downloaded locally.

## No French

Per project policy, French is never referenced, loaded, stored, or displayed. The `sensfr` field in the DGLAI dataset is not read. Entries from amawal that only have a French translation (no Arabic) are excluded.

## Dependencies

| Package | Version | Purpose |
|---------|---------|---------|
| `hono` | ^4 | HTTP framework (lightweight, fast) |
| `abjad-convert` | ^0.11 | Arabic ↔ Tifinagh transliteration |
| `tamazight` | ^1.0 | Latin ↔ Tifinagh transliteration |

Zero additional dependencies — Hono's middleware (CORS) is built-in.

## Future Ideas

- **Caching**: Add in-memory LRU cache for translate results
- **Rate limiting**: Add Hono middleware for production deployment
- **Health endpoint**: `GET /health` with DB status, uptime
- **Omnibot integration**: The bot queries this API for `/tifinagh` commands
- **Web UI**: Simple HTML search interface served from the same Hono app
